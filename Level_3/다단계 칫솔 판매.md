### 기록

1. **unordered_map을 사용하여 추천인의 인덱스 찾기**
    
    판매가 발생했을 때 추천인을 따라 위로 분배되는 구조이다. 각 판매자는 최대 한 명의 추천인만 가지므로 전체 구조는 부모 포인터를 가진 트리 형태로 볼 수 있다.
    
    따라서 각 사람의 이름을 인덱스로 매핑한 뒤, referral 배열을 이용해 부모를 추적하면서 이익을 분배하도록 구현했다.
    
    이를 위해 이름을 빠르게 인덱스로 변환할 수 있도록 unordered_map<string, int>를 했다. 추천인을 찾을 때마다 배열을 순회하면 비효율적이다. 
    
    이 문제는 알고리즘이 딱히 필요없는 문제라서 오랜만에 do-while 문을 사용해봤다.
    

### 제출 코드

```cpp
#include <string>
#include <vector>
#include <unordered_map>

using namespace std;

vector<int> solution(vector<string> enroll, vector<string> referral, vector<string> seller, vector<int> amount) {
    vector<int> answer(enroll.size(), 0);
    unordered_map<string, int> nameToIndex;
    
    for (int i = 0; i < enroll.size(); ++i) {
        nameToIndex[enroll[i]] = i;
    }
    
    for(int i = 0; i < seller.size(); i++){
        string currentSeller = seller[i];
        int profit = amount[i] * 100;
        do {
            int commission = profit / 10;
            answer[nameToIndex[currentSeller]] += profit - commission;
            currentSeller = referral[nameToIndex[currentSeller]];
            profit = commission;
        } while (currentSeller != "-" && profit > 0);
    }
    
    return answer;
}
```
