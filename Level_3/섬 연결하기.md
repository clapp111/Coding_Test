### 기록

1. **Kruskal (Union-Find)**
    
    이 문제는 각 섬을 정점, 다리 건설 비용을 간선의 가중치로 보는 그래프 문제이다. 모든 섬을 최소 비용으로 연결해야 하므로 최소 신장 트리(MST)를 구하는 문제로 볼 수 있다.
    
    가장 먼저 간선을 가중치의 오름차순으로 정렬하고, 간선을 순서대로 순회하면서 사이클이 형성되는지 확인해야한다.
    
    사이클 여부를 판단하기 위해 Union-Find를 사용한다. 간선의 양 끝 정점이 각각 속한 집합의 대표 노드(root)를 찾고, 두 대표가 다를 때만 간선을 선택하여 두 집합을 합친다.
    
2. **막혔던 점**
    - 처음에는 Union-Find에서 parent 배열의 값만 직접 비교하면 된다고 생각했다. 하지만 실제로는 단순 parent[x]가 아니라 find()를 통해 각 정점이 속한 집합의 대표 노드를 비교해야 했다.
    - Union-Find에서 부모 관계는 그래프의 방향을 의미하는 것이 아니라, 집합의 대표를 관리하기 위한 자료구조라는 점을 명확히 이해해야 한다.
    - Kruskal과 Prim은 강의에서 배울 때에도 힘들었고 다시 봐도 힘들었다. 그래서 그래프를 다루는 대표적인 알고리즘인 만큼 익혀서 나쁠건 없다고 생각한다.

### 제출 코드

```cpp
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

int getParent(vector<int>& parent, int x) {
    if (parent[x] == x) return x;
    return parent[x] = getParent(parent, parent[x]);
}

int solution(int n, vector<vector<int>> costs) {
    sort(costs.begin(), costs.end(), [](const vector<int>& a, const vector<int>& b){
        return a[2] < b[2];
    });

    vector<int> parent(n);
    for (int i = 0; i < n; i++) {
        parent[i] = i;
    }

    int answer = 0;
    int count = 0;
    for(const auto& cost : costs){
        int a = cost[0];
        int b = cost[1];
        int weight = cost[2];
        int parentA = getParent(parent, a);
        int parentB = getParent(parent, b);
        if(parentA != parentB){
            parent[parentB] = parentA;
            answer += weight;
            count++;
            if(count == n - 1) break;
        }
    }
    return answer;
}
```
