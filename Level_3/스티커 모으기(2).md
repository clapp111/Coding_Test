## 스티커 모으기(2) (Lv.3)

https://school.programmers.co.kr/learn/courses/30/lessons/12971

### 기록

1. **Dynamic Programming**
    
    이 문제는 스티커를 뜯을 떄 양 옆의 스티커를 사용할 수 없다는 점에서, 인접한 원소를 동시에 선택할 수 없는 DP 문제이다.
    
    기본 형태는 선형 배열에서의 `max(dp[i-1], dp[i-2]+sticker[i])` 점화식을 떠올릴 수 있다.
    
    다만 첫 번쨰 스티커와 마지막 스티커도 서로 인접해 있으므로 일반적인 선형 DP를 그대로 적용할 수 없다. 따라서 두 가지로 나누어서 해결해야 한다.
    
    첫 번째 스티커를 선택한 경우 vs 첫 번째 스티커를 선택하지 않은 경우
    
2. **막혔던 점**
    - 처음에는 두 번째 경우를 처리할 때, 단순히 두 번째 스티커를 반드시 선택한 경우라고 착각했다. 그래서 두 번째를 골랐다면 세 번째는 못 고르니 네 번째부터 탐색하는 방식으로 코드를 짰다. 효율성 테스트는 통과했지만, 이전의 정확성 테스트에서 틀리는 케이스가 있었다.
    - 즉, 두 번째 경우는 X, O 뿐만 아니라 X, X, O 와 X, X, X, O 같은 경우도 모두 포함해야 한다. 따라서 dp[1]만 확정하고 이후는 점화식으로 비교하면서 dp 배열을 채워나가면 된다.

### 제출 코드

```cpp
#include <iostream>
#include <vector>
using namespace std;

int solution(vector<int> sticker)
{
    int n = sticker.size();
    
    if(n == 1) return sticker[0];
    if(n == 2) return max(sticker[0], sticker[1]);
    
    vector<int> dp(n, 0);
    dp[0] = sticker[0];
    dp[1] = sticker[0];
    for(int i = 2; i < n - 1; i++){
        dp[i] = max(dp[i - 1], dp[i - 2] + sticker[i]);
    }
    int dp1 = dp[n - 2];

    fill(dp.begin(), dp.end(), 0);
    dp[1] = sticker[1];
    for(int i = 2; i < n; i++){
        dp[i] = max(dp[i - 1], dp[i - 2] + sticker[i]);
    }
    int dp2 = dp[n - 1];

    return max(dp1, dp2);
}
```
