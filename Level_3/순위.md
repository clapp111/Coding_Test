### 기록

1. **BFS + 인접리스트(그래프)**
    
    해당 문제는 순위를 확정 지을 수 있는 인원이 몇 명인지 알아내는 문제이다.
    
    순위를 확정되려면 그 선수보다 확실히 위에 있는 선수의 수와 아래에 있는 선수의 수를 모두 알아야 한다.
    
    그렇기 때문에 인접 리스트를 사용해서 그래프를 구현하고, 여기에서 BFS를 실행하면서 몇 명이 해당 방향(위, 아래)에 있는지 알 수 있다.
    
    내가 이긴 사람과 진 사람, 두 개의 인접 리스트를 만들어서 위의 과정을 실행하고, 이 값을 더한 값이 `총 인원 수 - 1` 이라면 순위가 확정된 사람이라고 볼 수 있다.
    
2. **회고**
    
    처음에는 단순히 직접 연결된 승패 관계만 세면 될 것 같았지만, 간접적인 승패 관계까지 포함해야 순위를 판단할 수 있다는 점이 핵심이었다. 오랜만에 BFS를 다루다 보니 방문 처리와 큐를 다루는 데 조금 버벅였다.
    
    다른 사람들의 풀이를 보니 Floyd-Warshall 알고리즘을 사용한 경우가 있었다. 정점 간 도달 가능 여부, 즉 승패 관계의 전이성을 처리하는 문제라서 잘 어울리는 방식이었다. 같은 문제도 그래프 탐색과 Floyd-Warshall처럼 여러 관점으로 접근할 수 있다는 점이 인상적이었다.
    

```cpp
#include <string>
#include <vector>
#include <queue>

using namespace std;

int bfs(int start, const vector<vector<int>>& graph, int n){
    vector<bool> visited (n + 1, false);
    queue<int> q;
    q.push(start);
    visited[start] = true;
    
    int count = 0;
    while(!q.empty()){
        int current = q.front();
        q.pop();
        for(int next : graph[current]){
            if(!visited[next]){
                visited[next] = true;
                q.push(next);
                count++;
            }
        }
    }
    return count;
}

int solution(int n, vector<vector<int>> results) {
    vector<vector<int>> win (n + 1);
    vector<vector<int>> lose (n + 1);

    for(const auto& result : results){
        int winner = result[0];
        int loser = result[1];
        win[winner].push_back(loser);
        lose[loser].push_back(winner);
    }

    int count = 0;
    for(int i = 1; i < n + 1; i++){
        int win_bfs = bfs(i, win, n);
        int lose_bfs = bfs(i, lose, n);
        if(win_bfs + lose_bfs == n - 1) count++;
    }
    
    return count;
}
```
