## 기지국 설치 (Lv.3)

https://school.programmers.co.kr/learn/courses/30/lessons/12979

### 기록

1. **수학 문제**
    
    오랜만에 수학적으로 해결하는 문제였다. 처음에는 어떤 알고리즘을 써야 할지부터 떠올렸지만, 이 문제는 이미 설치된 기지국들이 커버하는 구간을 기준으로 비어 있는 구간의 길이 만 구하면 해결할 수 있다.
    
    새 기지국 하나가 커버할 수 있는 범위의 길이는 `cover = 2 * w + 1` 이다. 아직 처리하지 않은 구간의 시작점을 `start` , 기존 기지국의 위치를 `station` 이라고 두면, 현재 기지국 왼쪽의 미커버 구간 길이는 `gap = station - w - start` 로 구할 수 있다.
    
    이 `gap` 을 새 기지국 하나가 덮을 수 있는 길이 `cover` 로 나누면 필요한 기지국 수가 나오며, 나머지가 있다면 하나를 더 설치해야 하므로 올림 나눗셈인 `(gap + cover - 1) / range` 를 사용하면 된다.
    
    기존 기지국들을 모두 순회한 뒤에도 `start <= n` 이라면, 마지막 기지국 오른쪽에도 미커버 구간이 남아 있는 것이므로 이 구간도 같은 방식으로 처리하면 된다.
    
2. **고쳐야 할 점**
    - 처음에는 빈 구간을 별도의 배열에 저장한 뒤 다시 순회하려고 했지만, 구간 길이를 구하는 즉시 정답에 반영하면 더 간단하고 효율적으로 풀 수 있었다.

### 제출 코드

```cpp
#include <iostream>
#include <vector>
using namespace std;

int solution(int n, vector<int> stations, int w)
{
    int answer = 0;
    int start = 1;
    int cover = 2 * w + 1;
    for(int station : stations){
        int gap = station - w - start;
        if(gap > 0){
            answer += (gap + cover - 1) / cover;
        }
        start = station + w + 1;
    }
    if(start <= n){
        int gap = n - start + 1;
        if(gap > 0){
            answer += (gap + cover - 1) / cover;
        }
    }
    return answer;
}
```
