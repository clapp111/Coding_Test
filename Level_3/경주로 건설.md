## 경주로 건설 (Lv.3)

https://school.programmers.co.kr/learn/courses/30/lessons/67259

### 기록

1. **다익스트라 + 우선 순위 큐**
    
    이 문제는 시작점에서 도착점까지의 최소 비용을 구하는 문제이므로 최단 경로 관점에서 접근할 수 있었다. 특히 이동 비용이 직진일 때는 100원, 코너일 때는 600원으로 달라지기 때문에 단순 BFS로는 해결할 수 없고 다익스트라 알고리즘이 적절하다고 판단했다.
    
    또한 이 문제는 같은 칸에 도착하더라도 어떤 방향으로 들어왔는지에 따라 다음 이동 비용이 달라진다. 따라서 `dist[x][y]`처럼 2차원으로 최소 비용을 저장하면 안 되고, `dist[x][y][dir]`처럼 방향까지 포함하여 상태를 관리해야 했다.
    
    탐색 구조는 BFS와 비슷하게 현재 위치에서 4방향으로 확장되지만, 다음에 탐색할 경로를 비용 기준으로 선택해야 한다는 점에서 다익스트라의 성격이 강하게 드러나는 문제였다.
    
2. **이전에 경험했던 문제의 오버랩**
    
    진행 방향에 따라서 경우를 나눠야 한다는 점에서 이전에 풀었던 [빛의 경로 사이클](https://school.programmers.co.kr/learn/courses/30/lessons/86052) 문제가 떠올랐다. 이미 풀었던 경험이 있는 만큼 방향을 어떻게 코드에 녹여낼지 떠올리기는 쉬웠다. 다만 이번 문제는 사이클 여부를 확인하는 문제가 아니라 최소 비용을 갱신해 나가야하는 문제였기 때문에, 탐색 방식에서는 분명한 차이가 있었다.
    
3. **고쳐야 할 점**
    
    이전에 비슷한 문제를 풀었던 기억 때문에 금방 해결할 수 있을 것이라고 생각했다. 하지만 실제로는 방향 상태를 정의하는 것과, 그 상태를 바탕으로 최소 비용을 갱신하는 탐색 로직을 구현하는 것은 다른 문제였다. 우선순위 큐를 사용해야 한다는 생각까지는 도달했지만, 이후 어떤 값을 비교하고 어떤 시점에 가지치기를 해야 하는지 정리하는 데 시간이 오래 걸렸다.
    
    특히 다익스트라에서는 같은 상태가 여러 번 우선순위 큐에 들어갈 수 있기 때문에, 이미 더 좋은 경로가 발견된 상태라면 현재 상태를 건너뛰는 처리가 필요하다.
    
    ```cpp
    int dir = top[3];
    if(cost > dist[x][y][dir])continue;
    for(int next_dir = 0; next_dir < 4; next_dir++){ ...
    ```
    
    예를 들어 우선순위 큐에 `(900, 2, 3, 1)`과 `(1500, 2, 3, 1)`이 함께 들어갈 수 있다. 이때 비용이 더 작은 900이 먼저 처리되어 `dist[2][3][1] = 900`이 되면, 이후에 나오는 1500은 더 이상 의미가 없는 정보가 된다. 이런 상태까지 계속 탐색하면 불필요한 연산이 늘어나기 때문에, 해당 조건을 통해 쓸모없는 경로를 가지치기해야 한다.
    

### 제출 코드

```cpp
#include <string>
#include <vector>
#include <queue>

using namespace std;

int solution(vector<vector<int>> board) {
    int n = board.size();
    vector<vector<vector<int>>> dist(n, vector<vector<int>>(n, vector<int>(4, 1e9))); // {x, y, dir}
    vector<vector<int>> move = {{-1, 0}, {0, 1}, {1, 0}, {0, -1}};

    priority_queue<vector<int>, vector<vector<int>>, greater<vector<int>>> pq; // {cost, x, y, dir}}
    pq.push({0, 0, 0, 1});
    pq.push({0, 0, 0, 2});
    dist[0][0][1] = 0;
    dist[0][0][2] = 0;

    while(!pq.empty()){
        auto top = pq.top();
        pq.pop();
        int cost = top[0];
        int x = top[1];
        int y = top[2];
        int dir = top[3];
        for(int next_dir = 0; next_dir < 4; next_dir++){
            int next_x = x + move[next_dir][0];
            int next_y = y + move[next_dir][1];
            if(next_x < 0 || next_x >= n || next_y < 0 || next_y >= n || board[next_x][next_y] == 1) continue;
            if(board[next_x][next_y] == 1) continue;
            
            int next_cost;
            if(next_dir == dir) next_cost = cost + 100;
            else next_cost = cost + 600;

            if(next_cost < dist[next_x][next_y][next_dir]){
                dist[next_x][next_y][next_dir] = next_cost;
                pq.push({next_cost, next_x, next_y, next_dir});
            }
        }
    }
    int min_cost = 1e9;
    for(int i = 0; i < 4; i++){
        min_cost = min(min_cost, dist[n-1][n-1][i]);
    }
    return min_cost;
}
```
