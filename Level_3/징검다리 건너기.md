
## 징검다리 건너기 (Lv.3)

### 기록

1. **슬라이딩 윈도우**
    
    해당 문제는 정확성과 효율성 테스트 각각 점수가 있는 문제이다. 또한 배열의 크기는 최대 200,000이며, 건널 수 있는 최대 칸수 역시 같다. 따라서 직관적으로 모든 경우를 계산하고 가지치기 하는 방법마저도 안먹힐 가능성이 높다고 생각했다. 
    
    이 문제의 핵심은 건널 수 있는 최대 칸수 $k$이다. 길이가 $k$ 인 연속 구간의 디딤돌들이 모두 0 이하가 되는 순간이다. 이 시점을 결정하는 값은 그 구간의 최댓값이고, 따라서 그 구간이 수용할 수 있는 최대 인원은 그 구간의 최댓값이라고 볼 수 있다.
    
    이 논리를 통해 배열의 모든 원소를 슬라이딩 하면서 탐색하면 답을 찾을 수 있다.
    
2. **multiset**
    
    추가적인 자료구조 없이 슬라이딩 윈도우를 구현한다면 각 구간에서 구간 내의 모든 원소를 비교하여 최대값을 찾고, 구간이 끝나면 최솟값을 갱신하는 방식으로 진행된다. 이렇게 되면 시간 복잡도가 $O(nk)$가 된다.
    
    multiset을 사용하여 새롭게 들어오는 원소를 추가하고, 윈도우의 범위에서 벗어난 이전 원소를 삭제하는 방식으로 구간 내의 모든 원소 찾아서 비교할 필요가 없앴다. 해당 구간 내에서 최댓값은 트리의 가장 오른쪽에 위치한 원소이므로 탐색하는데 $O(1)$의 시간 복잡도가 소모되며, 최종적으로는 ‘트리에서 값을 넣고 빼는 시간’ + ‘윈도우를 한 칸씩 옮기는 시간’ = $O(n \log k)$로 단축된다.
    
3. **고쳐야 할 점**
    - 처음엔 자료구조 없이 단순하게 구현했다. 그래서 정확성에서는 통과했지만, 효율성에서 절반이 시간 초과가 발생했다. 이 문제에서 슬라이딩 윈도우는 반드시 한 칸 씩 움직여야 하기 때문에 내부에서 값을 탐색하고 비교하는 방법을 개선했다.
    - 다른 사람들의 풀이를 보니깐 이분탐색으로 많이들 푸는 것 같다. 이 방식은 $O(n \log n)$이 걸리겠지만, 떠올리기 가장 간단한 방식이라고 생각했다. 또한 슬라이딩 윈도우를 사용하지만, 자료구조는 덱(dequeue)를 사용하면 $O(n)$으로 최적화 시킬 수 있다는 걸 알았다. 덱은 사용하기 어렵지만 알아두면 최적화 문제에서 요긴하게 쓰일 듯 싶다.
    

### 제출 코드

```cpp
#include <string>
#include <vector>
#include <set>

using namespace std;

int solution(vector<int> stones, int k) {
    int result = 200000000;
    int n = stones.size();
    multiset<int> s;
    for(int i = 0; i < k; i++){
        s.insert(stones[i]);
    }
    result = min(result, *s.rbegin());
    for(int i = k; i < n; i++){
        s.erase(s.find(stones[i-k]));
        s.insert(stones[i]);
        result = min(result, *s.rbegin());
    }
    return result;
}
```
